# round_charts — per-round price graphs

One image per resolved 5-minute round (`<window_start>_<UTC>.png`), generated by
`chart_capture.py` (a supervised service).

## What each chart shows
- **Up price line** (green) and **Down line** (faint red) over the 5-minute round,
  built from our own **1/second snapshots** (~300 points/round — high fidelity).
- **Polymarket's official price points** (black dots) overlaid from the public
  `prices-history` API, as **ground-truth validation** — if our line passes
  through their dots, our captured data is confirmed accurate.
- Title: round UTC time, official outcome (Up/Down), and BTC strike → final.

## Why this design (from deep research, task wrftip1ff)
The website's own chart API (`clob.polymarket.com/prices-history`) is **public but
coarse** — only ~3–5 points per 5-min round at its finest setting — and it returns
**empty once a round resolves** (the documented 12h-granularity limit). It also
needs a browser User-Agent (Cloudflare 403s otherwise). Our own snapshots are
~**60× finer** and always available, so they make the better line; the API points
serve purely as an independent ground-truth overlay, fetched **live near close**
(before resolution erases them). No headless browser, no ToS/bot risk.

A literal website screenshot (Playwright) was considered but rejected for now:
bot-protection/ToS is unverified and render-timing is finicky — and it would give
a *lower-fidelity* chart than this anyway.

## Notes
- **Backfill:** on startup `chart_capture.py` renders any settled round that lacks
  a chart (line only; official dots appear on rounds captured live going forward).
- Regenerate/backfill manually: `python chart_capture.py --once`.
- Images are git-ignored (regenerable); this README is tracked.
